Webservice voor de Centrale Activiteiten Database (CAD). De code komt zoveel mogelijk uit de JSP's en die van de subscibeEvent method uit de Struts actions. 1-op-1 copieren was echter niet mogelijk.

Globale structuur

  • ActiviteitenService: implementatie van de webservice
  • Beans (zoals Event, Natuurgebied etc.) die de data bevatten en door Axis naar SOAP messages worden geserializeerd.
  • BeanFactory: gebruikt door ActiviteitenService om de Beans te maken
    • ActiviteitenHelper: Helper voor code die is overgenomen uit JSP's etc.

Genereren client:

Dit kan bijvoorbeeld met Axis: 
wsdl2java -Nhttp://localhost:8340/natmm-internet/services/Activiteiten=nl.natuurmonumenten.activiteiten -o gen-src/ http://localhost:8340/natmm-internet/services/Activiteiten?wsdl
Voor de webservice is Axis 1.4 gebruikt. Axis 1.1 werd al gebruikt in de code maar is wel ge-upgrade naar 1.4. Dit maakt meer complient WSDL dan versie 1.1. De webservice wordt geconfigureerd in server-config.wsdd die in WEB-INF staat.
